Class Overview

Course Content

Layout

The first portion of this class focuses on the layout and styling websites. In this portion of the class we will learn how to use HTML5 and CSS to create layouts for pages and style those layouts. We will write code that is conformant with current standards as published by the W3C.

Interactivity

The second portion of this class will focus on making interactive websites. To accomplish this we will use JavaScript to implement client side interactions. A user will be able to interact with a web page and see changes in real time. Examples might include implementing a simple calculator or periodically checking for new news stories and appending them to the page as the appear.

This will also include more complex interactions using Ajax along with an introduction to the storage of data on the client computer.

Server Side Interaction

The third portion of this class will focus on creating dynamic web pages on the server side. This means the content sent to the client from the server will change based on the state of the server. This could include things like allowing a user to save a to-do list and later retrieve it from a different computer or letting users asynchronously play a game of checkers against each other. We will accomplish this using PHP as our server side scripting language.

This portion of the class will also teach very basic database interaction and we will learn how to save and later retrieve data from a MySQL database to be used in the rendering of web pages.

Design Considerations

This portion of the class will look at the topics of security, scalability and usability in the context of web applications. We will learn about these topics while implementing a final project which should use the best practices from each area to produce an efficient, usable and scalable web application.

Student Expectations

Prior Knowledge

Students are expected to know the following:

Basic data types
This list of data types should be familiar or at least understandable after reading the type description.
Common object types
You should know what a string is. You should know if there is a difference between a string and an array. You should know the difference between a string in C and a string object in C++
Complex data types
You should know what an array is, what a struct is and what, if any, differences there are between them.
Flow control primitives
You must be very comfortable with for, while, if/else, switch and do/while. You should know how they work, you should know what i is equal to at the completion of a loop if the condition says for(i=0, i<5, i++), is it 5 or is it 6?
Scope
No matter where I declare int foo in your code, you should be able to figure out if any other arbitrary spot in your code can access that variable. You should know what public and private functions are and what happens (and what it means) when variables go out of scope.
Problem solving
If a language does not provide you a tool to do something, but you need that tool to make progress, you should be comfortable making it yourself. You should be comfortable making helper functions, even when the requirements don't call for it.
Organization and documentation
You should know that all of your source code probably does not belong in a single file and that every public function should have comments.
Things not on this list
This is not a comprehensive list, but it is a good start. If you don't know any of these topics, consider reviewing them before the class begins.

Communication

Read this guide on online communication. Follow every link in the section on asking good questions and giving good answers. You may be graded on the quality of your discussion. You need to communicate with clients and coworkers in the real world, you need to communicate with your fellow students here. You learn by finding the limits and holes in your knowledge. The best way for you to find them is to talk with others about the topics at hand.

Code quality

The easiest way to sum this up is that every program must be syntactically perfect. There are tools you can use to check the syntax of your program. The only exception to this rule is if I specifically allow a warning or notification in one of these checkers. Run time errors are unacceptable as well. In no situation should your program output a standard PHP error or throw an error on the console. If there is a problem or the user put the site into an invalid state, the error should be handled in a way as to gracefully notify the user that there was a problem.

In terms of style, all code should conform to the Google HTML5 and CSS style guide, the Google JavaScript style guide and the Pear PHP style guide. There are several programs you can download to check if your file conforms to these guidelines.
